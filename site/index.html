Familiar APIs

Write JavaScript you already know.

Handlers look like Bun or the web platform. The process underneath is Common Lisp.

hello.ts 
const project: string = "Clun";
const runtime = "Common Lisp";

console.log(`${project} runs on ${runtime}`);
const server = Clun.serve({
  port: 3000,
  fetch: async (request: Request) => {
    return Response.json({ ok: true });
  },
});

console.log(server.url);
test("math stays honest", () => {
  expect(2 + 2).toBe(4);
});
const file = Clun.file("message.txt");

Clun.write(file, "hello from Clun\n")
  .then(() => file.text())
  .then((text) => console.log(text));

Implementation

Common Lisp all the way down.

Not a thin wrapper around Node. Not a foreign bridge to V8. ANSI Common Lisp + SBCL + vendored Lisp libraries — CFFI shortcuts for Clun itself are rejected.

  1. 01
    From-scratch engine

    Lexer, parser, object model, modules, async, and runtime APIs live in this repository.

  2. 02
    Vendored and pinned

    A fresh source build pulls no Quicklisp packages and no runtime npm for Clun itself.

  3. 03
    Purity is checked

    CI runs make purity so foreign-library slips cannot land quietly.

  4. 04
    TLS is pure Lisp too

    HTTPS for installs and clients uses Clun’s own TLS 1.2/1.3 stack (ironclad + pure-tls). CI runs protocol, alert, and OpenSSL interop suites via make test-tls — not an untested path. Full third-party WebPKI audit is still a separate bar (detail on GitHub).
